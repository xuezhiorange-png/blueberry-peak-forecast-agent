# Codex 总控提示词

将本仓库交给 Codex 后，先发送以下提示词：

> 你正在开发“蓝莓峰值预测 Agent”。先完整阅读 `AGENTS.md`、`README.md`、`CODEX_TASKS.md` 和 `docs/`。产品默认用户只提交农场位置和各品种定植亩数，系统必须自动从历史数据库和气象数据推断亩产、有效商品果率、成熟曲线、春节采摘实现率及不确定性，并输出1—4月逐日P50/P80/P90、单日操作峰值、连续3日持续峰值、产季曲线、详细计算说明、可信度和规则化建议。加工厂建筑面积不得作为预测特征。普鲜、普青、普冻、废果和巴松加工厂按配置在分析层剔除。LLM不得自行计算或编造吨数，所有数值必须来自确定性服务。先执行 `CODEX_TASKS.md` 的任务0，不要跨阶段实现；先写测试，完成后运行 lint、type check 和 pytest，并报告变更、测试结果、风险和下一任务建议。

## 每个后续任务的固定追加要求

> 不允许使用预测日期之后的数据，不允许用最终产季总量进行真实提前预测回测。任何自动推断参数都必须保存来源层级、样本量、产季数、版本和历史误差。普通用户缺少高级数据时不得拒绝规划预测，应使用版本化分层先验并扩大P80/P90区间。不得在代码中硬编码弥勒Dx等具体经验参数。
