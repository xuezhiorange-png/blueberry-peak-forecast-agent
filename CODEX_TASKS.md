# 可直接交给 Codex 的开发任务

建议每次只执行一个阶段，完成测试并提交后再进入下一阶段。

## 任务 0：初始化工程

**提示词**

> 阅读 README、AGENTS.md 和 docs。初始化 Python 3.12 FastAPI 工程、PostgreSQL、SQLAlchemy 2、Alembic、pytest、ruff、mypy 和 Docker Compose。不得实现占位式假接口；健康检查必须真实检查数据库连接。完成后运行测试并报告变更文件。

**验收**
- `docker compose up -d db` 可运行。
- `/health/live` 和 `/health/ready` 正常。
- CI 执行 lint/type/test。

## 任务 1：数据库与主数据

> 按 `docs/03_database_design.md` 和 `sql/schema.sql` 建立 ORM 与 Alembic 迁移。实现 season、factory、farm、subfarm、variety、grade、holiday 主数据 CRUD。唯一约束和外键必须完整。

## 任务 2：历史 XLS 导入

> 按 `docs/04_historical_import.md` 实现旧版 `.xls` 导入器。支持多 Sheet、字符串日期和 Excel 日期；按配置归一化品种和加工厂名称；默认剔除规则只作用于 curated 层，raw 层完整保留。实现文件 SHA256、行指纹、幂等导入、质量报告和 dry-run。

**验收**
- 能导入两个历史文件。
- 重复执行不增加事实行数。
- 输出总行数、有效重量、剔除重量、无效日期、未知工厂、重复行统计。

## 任务 3：每日事实层与峰值定义

> 聚合 `fact_receipt_raw` 到 `fact_receipt_daily`。实现单日峰值、连续3日中位持续峰值、连续3日均值峰值、峰值日期、峰值集中度、品种/农场/分场 HHI。春节窗口作为标签，不直接删除原始日序列。

## 任务 4：基线模型与历史复现

> 实现三个基线：上一季峰值、当季预计总量×上一季集中度、总量+品种/农场/分场集中度 Ridge 模型。复现现有参考回测，允许因数据库清洗差异有小幅偏差。生成逐工厂误差表。

**验收目标**
- 最优静态基线跨产季 MAPE 应接近 13%—16%。
- 若明显优于 10%，优先检查数据泄漏。

## 任务 5：极简输入、位置解析与自动参数库

> 实现默认用户入口：只提交农场位置和各品种定植亩数。实现地址标准化、经纬度/海拔解析、农业气候区映射、相似历史样本检索，以及“位置×品种”亩产、商品果率、成熟曲线和采摘实现率参数库。自动参数必须记录来源层级、样本数、产季数、版本和回测误差。不得硬编码弥勒Dx等业务参数。

**验收**
- 仅输入位置和品种亩数即可创建预测任务；
- 同一输入和同一数据版本可复现；
- 无本农场历史时能分层回退并标低可信度；
- 返回自动参数及其P80区间和来源。

## 任务 6：产量计划与物候数据

> 实现 farm-season-variety 计划录入：面积、预计亩产、商品果率、树龄、修剪日期、花期、首采日期、预计总商品果量及版本。所有计划支持版本和生效时间，回测时只能读取预测当日已经存在的版本。

## 任务 7：天气数据与物候时间轴

> 实现 provider 接口、CSV 导入和网格/站点映射。计算 7/14/21 天有效积温、辐射、降雨、连续阴雨、最低温和昼夜温差。基温通过训练集搜索，不能预先固定。

## 任务 8：自然成熟曲线模型

> 以分场×品种为粒度，训练区域—品种共享的平滑成熟曲线。输入预计有效商品果总量、物候时间、海拔、设施和修剪/花期偏移；输出每日自然成熟量及 P50/P80/P90。优先使用 GAM/样条和部分池化，避免小样本黑箱过拟合。

## 任务 9：春节、用工与积压状态模型

> 实现成熟库存状态：新增成熟量、未采积压、每日采摘能力、实际到厂量。采摘能力使用实际人数×人均能力×天气效率。春节作为影响人员可用率的变量。实现节后积压释放，不得直接把春节窗口全部删除。

## 任务 10：残差模型

> 使用 HistGradientBoostingRegressor 或 LightGBM 修正结构模型残差。特征只允许使用预测时点已知信息；实现特征审计，阻止未来信息泄漏。

## 任务 11：滚动预测与回测

> 实现每周/每日滚动预测。回测节点至少包含2月底、3月15日、3月31日、4月7日。评价峰值吨数 APE、日期 MAE、日曲线 WMAPE、P80/P90 覆盖率、高峰预警召回率和假警率。

## 任务 12：预测 API 与前端

> 实现极简农场预测、情景模拟、模型解释、数据质量、回测结果 API。默认表单只含位置和品种亩数；结果必须包含逐日P50/P80/P90、峰值卡片、产季曲线、品种贡献、自动参数、计算说明、准确性和建议。高级参数放在可选折叠区。

## 任务 13：Agent 编排层

> 增加工具调用式 Agent。Agent 只能调用确定性服务：resolve_location、infer_parameters、forecast_daily_curve、forecast_peak、simulate_scenario、run_backtest、explain_forecast、generate_recommendations。普通用户只给位置和品种亩数时必须完成规划预测；高级数据缺失时使用版本化先验并扩大区间，不得随意编造。所有回答附 model_run_id、参数来源和可信度。

## 任务 14：生产化

> 增加权限、审计日志、备份、模型注册、监控、漂移检测、任务调度和告警。预测性能下降或输入缺失时自动回退到基线模型，并标记低可信度。
