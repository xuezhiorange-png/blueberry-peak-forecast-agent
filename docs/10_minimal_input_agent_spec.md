# 极简输入峰值预测 Agent 规范

## 1. 产品定位

用户只需要提交两类信息：

1. 农场位置；
2. 各品种定植亩数。

系统不得把预计亩产、商品果率、成熟曲线、气象参数、春节影响或采摘效率作为普通用户的必填项。这些参数由历史数据库、位置气候数据和模型自动推断；高级用户可在“参数校正”中覆盖默认值。

## 2. 用户最小输入

### 农场位置

至少支持以下任一形式：

- 省/州（市）/县/乡镇文字地址；
- 地图选点；
- 经纬度。

系统自动补充：

- 经纬度；
- 海拔；
- 行政区域；
- 农业气候区；
- 最近气象站或网格；
- 历史相似农场集合。

### 品种与面积

用户逐行录入：

| 品种 | 定植亩数 |
|---|---:|
| Dx | 700 |
| D12 | 300 |

面积为农场生产规模变量。加工厂建筑面积不得进入预测模型。

## 3. 系统自动推断参数

每个“位置×品种”自动推断：

- 1—4月预计亩产分布；
- 有效商品果率分布；
- 预计有效商品果总量；
- 首采日期分布；
- 成熟曲线形状和宽度；
- 气温、积温、辐射和降雨修正；
- 春节期间采摘实现率；
- 节后积压释放强度；
- 历史异常峰值概率。

推断优先级：

1. 同农场、同品种历史；
2. 同乡镇/相近海拔、同品种历史；
3. 同县/同农业气候区、同品种历史；
4. 云南同品种总体先验；
5. 品种资料先验，仅作为无历史数据时的低可信度回退。

系统必须显示每个参数的来源、样本数、覆盖产季数和历史回测误差。

## 4. 计算流程

### 4.1 位置解析

将文字位置解析为标准坐标和农业气候区，并检索历史相似样本。相似度至少考虑：

- 水平距离；
- 海拔差；
- 纬度；
- 1—4月温度与积温；
- 日照/辐射；
- 降雨；
- 品种一致性。

### 4.2 产量规模推断

对品种 v：

\[
V_v=A_v\times \hat q_{l,v}\times \hat r_{l,v}
\]

- \(A_v\)：用户填写的定植亩数；
- \(\hat q_{l,v}\)：位置 l、品种 v 的预计亩产；
- \(\hat r_{l,v}\)：有效商品果率；
- \(V_v\)：预计有效商品果总量。

\(q\) 和 \(r\) 必须作为概率分布参与蒙特卡洛或分位数模拟，不得只使用一个无误差的固定值。

### 4.3 自然成熟曲线

\[
M_{v,t}=V_v\times G_{z,v}(\tau_t-\delta_{l,v})
\]

其中 \(G\) 来自历史“农业气候区×品种”的归一化曲线，\(\tau_t\) 使用逐日气象驱动物候时间。

### 4.4 春节、天气与采摘实现

\[
Y_t=\min(H_t,M_t+\lambda_t B_{t-1})
\]

\[
B_t=\max(0,B_{t-1}+M_t-Y_t)
\]

在用户未提交人员计划时，\(H_t\) 使用同区域历史采摘实现率和春节阶段分布推断，并提高不确定性区间。春节不直接删除数据。

### 4.5 多品种叠加

\[
Y_t=\sum_vY_{v,t}
\]

模型必须保留每个品种逐日贡献，计算品种峰值重叠度，并解释总峰值由哪些品种叠加形成。

## 5. 必须输出

### 5.1 逐日产能预测

从产季开始至4月30日，每日输出：

- P50预测到果量；
- P80区间；
- P90区间；
- 各品种贡献；
- 自然成熟量；
- 采摘实现量；
- 估计积压量；
- 主要天气/春节标签。

### 5.2 峰值产能分析

同时输出：

- 单日操作峰值及日期；
- 连续3日持续峰值及日期；
- P80/P90峰值；
- 峰值持续天数；
- 峰值前后7天累计量；
- 主导品种及贡献率；
- 峰值形成原因；
- 与历史相似农场对比。

### 5.3 产季产能曲线

展示：

- 总到果曲线；
- 品种堆叠曲线；
- P50/P80/P90带状区间；
- 春节、降雨、升温过程标记；
- 峰值日期与高负荷区间。

### 5.4 详细计算说明

报告至少解释：

- 用户输入；
- 自动识别的位置和海拔；
- 使用的历史样本；
- 自动推断的亩产和商品果率；
- 各品种成熟曲线来源；
- 天气修正；
- 春节与采摘实现假设；
- 单日峰值与持续峰值差异；
- 模型版本和回测表现；
- 当前预测的主要不确定性。

### 5.5 建议

建议必须由确定性规则和情景模拟生成，不由LLM凭空编造。至少覆盖：

- 推荐持续处理能力；
- 推荐收货/暂存/预冷瞬时能力；
- 高峰期班次建议；
- 春节人员准备；
- 品种错峰或修剪批次建议；
- 是否需要跨厂分流；
- 哪些额外数据最能提升准确性。

## 6. 两种运行模式

### 规划模式

仅使用“位置＋品种亩数”。适用于新农场或产季前规划。系统自动推断全部参数，并输出较宽的P80/P90区间。

### 动态模式

产季开始后，自动融合：

- 累计实际入库；
- 最新天气；
- 物候记录；
- 采摘人数；
- 最新产量计划。

每日或每周更新曲线，区间应逐步收窄。

## 7. 可信度等级

- 高：同农场同品种≥2个产季，位置和气象完整，滚动回测达标；
- 中：同气候区同品种样本充分，但缺少本农场历史；
- 低：同品种历史少、位置解析粗、亩产和商品果率主要依靠省级先验。

不得只给“高/中/低”，还要给出：样本数、历史MAPE、日期MAE、P90覆盖率和关键缺失项。
